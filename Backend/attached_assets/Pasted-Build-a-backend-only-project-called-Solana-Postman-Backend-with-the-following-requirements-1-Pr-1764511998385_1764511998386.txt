Build a backend-only project called “Solana Postman Backend” with the following requirements:

1. Project Purpose

Create a FastAPI backend that can serve as the engine for a future Solana Postman tool.

The backend must provide APIs that allow a client to:

Input RPC URL

Input Program ID

Auto-fetch Anchor IDL (if it exists)

Parse the IDL and return available methods/instructions

Generate input schemas for the methods

Build transactions

Simulate transactions

Serialize arbitrary instruction data for non-IDL programs (Byte Packer mode)

This must be backend only — no frontend, no HTML templates, no static files.

2. Backend Stack Requirements

Use Python + FastAPI.

Required dependencies:

fastapi

uvicorn

httpx or aiohttp

solana (solana-py)

pydantic v2

base58

borsh-construct (optional but recommended for IDL-based serialization)

Use async everywhere for performance.

3. Required Endpoints

Implement the backend in modular routers.
Use this structure:

backend/
    app/
        main.py
        routers/
            idl.py
            simulate.py
            instructions.py
            accounts.py
        core/
            solana_rpc.py
            idl_loader.py
            byte_packer.py
            tx_builder.py
        models/
        utils/
    requirements.txt
    TODO.md

Required APIs:
A. IDL / Program Introspection
GET /idl/{program_id}


Fetch IDL account

If Anchor IDL is found: return parsed JSON

If not found: return error

GET /idl/{program_id}/methods


Parse IDL

Return list of instructions + argument schemas

B. Instruction Builder

For programs without IDL.

POST /instruction/pack


Body example:

{
  "layout": [
    {"type": "u8", "value": 1},
    {"type": "u64", "value": 1000},
    {"type": "pubkey", "value": "<pubkey>"}
  ]
}


Return:

{
  "buffer_hex": "01ff0a9b..."
}

C. Transaction Builder
POST /tx/build


Takes:

program_id

accounts

instruction data (bytes or hex)

Returns:

compiled transaction (base64)

D. Simulation
POST /tx/simulate


Takes:

base64 transaction

RPC URL

Returns:

logs

error if any

compute units

balance changes if possible
