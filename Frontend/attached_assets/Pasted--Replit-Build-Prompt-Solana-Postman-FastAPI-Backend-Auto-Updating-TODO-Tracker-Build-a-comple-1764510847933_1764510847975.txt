✅ Replit Build Prompt: Solana Postman (FastAPI Backend) + Auto-Updating TODO Tracker

Build a complete project called “Solana Postman” with the following requirements:

1. Project Goal (Summary)

Create a zero-friction Solana Client where a user can:

Enter an RPC URL

Enter a Program ID

Automatically fetch the Anchor IDL if it exists

Display all methods in the IDL

Auto-generate method input fields

Simulate transactions (dry-run)

Send transactions

For programs without IDL:

Build instructions manually with a visual Byte Packer

This tool should behave like a "Solana Postman".

2. Three Modes To Support
A. Anchor Auto-Magician Mode

Input: Program ID

Fetch IDL account

Load JSON IDL

Auto-generate input schema

Show methods

Allow simulate + send

B. Instruction Builder Mode

Visual byte packer

User selects types: u8, u16, u32, u64, Pubkey, string, bytes

Show real-time hex buffer preview

C. Transaction Simulator Mode

Perform Solana simulateTransaction

Parse logs into readable text

Show:

balance changes

compute units used

return data

3. Backend Requirements
Backend stack: Python + FastAPI

Use:

FastAPI for the backend

Pydantic v2 for request/response validation

HTTPX / aiohttp for async RPC calls

Solana Python SDK (solana-py) for:

building instructions

transaction simulation

signing (optional)

The backend will:

Fetch and parse Anchor IDLs

Query program accounts

Serialize instruction data

Simulate transactions

Build/send raw transactions

Use async everywhere for speed.

4. Frontend Requirements

Use:

React (Vite + TypeScript)

TailwindCSS

Clean UI

Frontend Features:

RPC URL input

Program ID input

Auto-loaded IDL → Methods dropdown

Dynamic form generator

Instruction Builder (hex buffer view)

Simulation viewer

Pretty logs + error explanations
